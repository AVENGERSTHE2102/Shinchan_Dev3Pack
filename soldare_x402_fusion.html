The full flow — in plain English
1. Aditya creates a dare — types "I dare you to cold call a random number and say you won a prize", attaches $2 USDC, gets a shareable link: soldare.xyz/d/abc123
2. Friend opens the link on their phone — sees the dare, the bounty amount, and a countdown. No wallet needed to view. Clicks Accept.
3. Friend records/photos the proof — uploads it or pastes an X/Instagram link. Clicks "I did it".
4. Aditya gets a notification — sees the proof, clicks "Approve". The x402 server fires a USDC transfer via the Solana x402 facilitator. Friend's wallet gets $2 in under a second.
5. Expired or refused? Anchor program's reclaim instruction fires after 48h. SOL goes back. Automatic. No dispute.
Architecture — what x402 does vs what Anchor does
Anchor program (Rust)
Holds the dare state on-chain: creator, recipient, bounty, status, deadline
Locks SOL/USDC in a PDA vault when dare is created
Enforces expiry via Clock::get() sysvar
Emits events frontend listens to for real-time UI updates
~200 lines Rust3–4 instructions
x402 layer (TypeScript)
Dare page is an x402-protected route — viewing it costs $0 but claiming the bounty triggers a payment response
When creator approves, server sends x402 USDC payout to friend's wallet instantly
Uses @x402/next middleware — 5 lines of config

The Anchor program — 4 clean instructions
struct DareAccount {
  creator:   Pubkey,   // who sent the dare
  recipient: Pubkey,   // who received it (their wallet)
  bounty:    u64,      // lamports locked in PDA
  dare_hash: [u8; 32], // hash of dare text — immutable proof
  expires_at:i64,      // unix timestamp
  status:    DareStatus,// Open | Accepted | Done | Expired
}

// create_dare → creator deposits SOL into PDA, sets expiry
// accept_dare → recipient signs, status = Accepted
// approve → creator signs approval → x402 payout fires in frontend
// reclaim → after expiry, creator reclaims SOL

The Anchor program is the trust layer. x402 is the payment delivery layer. They work together — one enforces the rules, the other handles the money movement. That architectural split is itself impressive to judges.
